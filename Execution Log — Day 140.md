**Day 140 — Execution Log | Date : 10/05/2026**



**Session Type:** Core Session



**Topic:** Context Switching



**Concept:**

Context switching occurs when the operating system saves the state of one task and switches CPU execution to another task.



**Key Learnings:**

* Context switching enables multitasking
* CPU cores execute one task at a time
* Switching has performance overhead
* Excessive threads can reduce efficiency



**Core Insight:**

Concurrency improves multitasking, but excessive context switching can harm performance.



**Outcome:**

* Strengthened understanding of multitasking internals
* Connected scheduling with performance overhead
* Improved reasoning about efficient concurrency design
