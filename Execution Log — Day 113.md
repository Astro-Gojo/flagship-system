**Session Type:** Core Session



**Topic:** Deadlock (Structural Understanding)



**Concept:**

Deadlock occurs when a set of threads are waiting for each other in a circular dependency, causing permanent blocking.



**4 Necessary Conditions:**

* Mutual exclusion
* Hold and wait
* No preemption
* Circular wait



**Key Learnings:**

* Deadlock is caused by structural design issues
* It only occurs when all four conditions are present
* Breaking any one condition prevents deadlock



**Core Insight:**

Deadlock is not random; it is a predictable outcome of specific system conditions.



**Outcome:**

* Understood true cause of deadlock
* Learned how to think about prevention
* Strengthened failure analysis in systems
