**Topic:** Stateless vs Stateful Services



**Concepts Learned:**

* Stateful Services
* Stateless Services
* Request Independence
* Fault Tolerance



**Key Insights:**

* Stateless services allow any server to handle any request.
* Shared storage holds persistent state while application servers remain lightweight.
* Stateless architectures simplify scaling, recovery, and load balancing.



**Systems Engineering Principle:**

Systems become more scalable when computation is independent and persistent state is managed separately.
