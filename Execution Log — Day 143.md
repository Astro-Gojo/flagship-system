**Session Type:** Core Session



**Topic:** Readers–Writers Problem



**Concept:**

The readers–writers problem manages shared data access by allowing multiple simultaneous readers while ensuring writers have exclusive access.



**Key Learnings:**

* Reading operations can safely occur concurrently
* Writing operations require exclusive access
* Synchronization must balance safety, fairness, and efficiency
* Improper balancing can cause starvation problems



**Core Insight:**

Efficient synchronization allows safe parallel access while restricting only conflicting operations.



**Outcome:**

* Strengthened understanding of optimized synchronization
* Connected fairness with concurrent access control
* Improved reasoning about scalable shared-resource systems
