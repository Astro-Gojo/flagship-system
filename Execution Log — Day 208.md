**Session Type:** Core Session



**Topic:** Event-Driven Architecture (EDA)



**Concepts Learned:**

* Event-Driven Architecture
* Publishers
* Subscribers
* Event Bus
* Loose Coupling



**Key Insights:**

* Publishers emit events without knowing who consumes them.
* Subscribers independently react to events.
* EDA improves scalability, maintainability, and fault isolation.
* Message queues and events complement each other but serve different purposes.



**Systems Engineering Principle:**

Loosely coupled systems evolve more easily because components communicate through events rather than direct dependencies.



**Progression:**

Message Queues → Event-Driven Architecture → Loose Coupling
