**Day 136 — Execution Log | Date : 06/05/2026**



**Session Type:** Core Session



**Topic:** Process vs Thread



**Concept:**

A process is an independent running program, while a thread is a lightweight execution unit inside a process.



**Key Learnings:**

* Processes have separate memory and resources
* Threads share memory/resources within a process
* Threads enable efficient concurrency
* Shared memory increases synchronization risks



**Core Insight:**

Processes provide isolation, while threads provide lightweight parallel execution and responsiveness.



**Outcome:**

* Strengthened understanding of modern application structure
* Connected concurrency concepts with program architecture
* Improved system-level reasoning about execution models
