**Session Type:** Core Session



**Topic:** Mutex vs Semaphore



**Concept:**

Mutex provides mutual exclusion for protecting shared data, while semaphores control access to limited resources using counters.



**Key Learnings:**

* Mutex allows only one thread at a time
* Semaphore can allow multiple threads simultaneously
* Mutex is ownership-based
* Semaphore is counter/resource-based



**Core Insight:**

Different synchronization tools solve different concurrency coordination problems.



**Outcome:**

* Strengthened understanding of synchronization mechanisms
* Connected concurrency problems with practical solutions
* Improved reasoning about safe resource management
