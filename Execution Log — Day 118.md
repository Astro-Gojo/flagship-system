**Day 118 — Execution Log | Date : 18/04/2026**



**Session Type:** Core Session



**Topic:** Dining Philosophers Problem



**Concept:**

A classic synchronization problem demonstrating deadlock and starvation in systems with multiple resources and competing threads.



**Key Problems:**

* Deadlock due to circular wait
* Starvation due to unfair access



**Solutions Learned:**

* Resource ordering (break circular wait)
* Limiting concurrent access
* Asymmetric resource acquisition



**Core Insight:**

Deadlock arises from circular dependencies; breaking the cycle ensures system progress.



**Outcome:**

* Applied multiple system concepts in one scenario
* Strengthened understanding of real-world synchronization issues
* Improved system design thinking
