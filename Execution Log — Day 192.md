**Session Type:** Micro Session



**Topic:** Sticky Sessions



**Concepts Learned:**

* Session Affinity
* User Continuity
* Session Dependency



**Key Insights:**

* Keeping a user on the same server simplifies session management.
* Sticky sessions reduce resilience because session data becomes tied to one server.
* Modern systems often externalize session storage to avoid this dependency.



**Systems Engineering Principle:**

Convenience often introduces coupling; reducing coupling improves resilience.



**Progression:**

Load Balancing → Health Checks → Sticky Sessions
