**Day 121 — Execution Log | Date : 21/04/2026**



**Session Type:** Core Session



**Topic:** Context Switching



**Concept:**

Context switching is the process of saving and restoring thread state to enable multiple threads to share a single CPU.



**Key Components of Context:**

* Program counter
* CPU registers
* Execution state



**Key Learnings:**

* Enables concurrency on single-core systems
* Not free — introduces CPU overhead
* Excessive switching reduces performance



**Core Insight:**

Concurrency is achieved through rapid context switching, not simultaneous execution.



**Outcome:**

* Understood mechanism behind concurrency
* Learned performance implications of switching
* Strengthened system-level execution understanding
