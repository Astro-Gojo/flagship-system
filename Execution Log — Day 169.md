**Session Type:** Micro Session



**Topic:** Throughput vs Latency



**Concepts Learned:**

* Latency measures response time.
* Throughput measures overall system capacity.
* Improving one does not necessarily improve the other.



**Key Insights:**

* Systems must balance speed per request against total work completed.
* High throughput may sometimes increase latency.



**Systems Engineering Principle:**

Performance consists of multiple dimensions, and optimization requires balancing trade-offs.



**Progression:**

Performance intuition expanded from:

* indexing

→ response time

→ capacity

→ throughput-latency trade-offs
