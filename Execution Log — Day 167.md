**Session Type:** Core Session



**Topic:** Load Balancing \& Request Distribution



**Concepts Learned:**

* Load balancing distributes requests across multiple servers.
* Proper distribution improves scalability and resource utilization.
* Round Robin distributes requests sequentially.
* Least Loaded routes traffic to the least busy server.
* Load balancers improve availability by rerouting traffic around failures.



**Key Insights:**

* Additional hardware does not automatically improve performance.
* Poor workload distribution can leave bottlenecks unchanged.
* Availability improves when traffic can be redirected after failures.
* Load balancing is another example of distributing work to remove bottlenecks.



**Systems Engineering Principle:**

Scalable systems require both:

* sufficient resources
* effective workload distribution

Performance gains come not only from adding capacity, but from utilizing that capacity intelligently.



**Progression:**

Systems understanding expanded from:

* distributed caching

→ request distribution

→ load balancing strategies

→ fault-tolerant routing

→ bottleneck elimination
