**Day 124 — Execution Log** **| Date : 24/04/2026**



**Session Type:** Core Session



**Topic:** CPU-bound vs I/O-bound Tasks



**Concept:**

CPU-bound tasks use CPU most of the time, while I/O-bound tasks spend significant time waiting for input/output operations.



**Key Learnings:**

* CPU-bound tasks have long CPU bursts
* I/O-bound tasks frequently wait
* Switching tasks during I/O wait improves CPU utilization
* Mixing both types improves system performance



**Core Insight:**

Efficient systems maximize CPU usage by scheduling tasks based on their behavior.



**Outcome:**

* Understood workload characteristics
* Connected task behavior with scheduling strategies
* Strengthened performance-aware system thinking
