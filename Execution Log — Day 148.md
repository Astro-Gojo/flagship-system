**Session Type:** Core Session



**Topic:** Memory Consistency / Memory Ordering



**Concept:**

Memory consistency and memory ordering define how memory updates become visible across threads and CPU cores in concurrent systems.



**Key Learnings:**

* Multi-core CPUs use caches and reordering optimizations
* Memory visibility may differ between threads
* Correct concurrency depends on proper ordering and visibility guarantees
* Atomic operations and synchronization primitives help enforce consistency



**Core Insight:**

Concurrent correctness depends not only on execution, but also on when and how updates become visible across threads.



**Outcome:**

* Strengthened understanding of hardware-level concurrency behavior
* Connected CPU optimizations with synchronization complexity
* Improved reasoning about visibility-related concurrency bugs
