**Day 109 — Execution Log | Date : 09/04/2026**

**Session Type:** Core Session



**Topic:** Starvation (Fairness in Concurrent Systems)



**Concept**:

Starvation occurs when a thread waits indefinitely while other threads continue executing.



**Key Learnings:**

* Starvation is caused by lack of fairness in scheduling
* Semaphore and mutex ensure correctness but NOT fairness
* A system can function correctly but still behave unfairly



**Important Distinction:**

* Deadlock → no progress
* Starvation → partial progress (some threads blocked forever)



**Core Insight:**

Correctness does not guarantee fairness in system behavior.



**Outcome:**

* Understood limitations of synchronization mechanisms
* Learned to evaluate systems beyond correctness
* Improved real-world system thinking
