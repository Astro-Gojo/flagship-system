**Session Type: Core Session**



**Topic:** Readers–Writers Problem



**Concept:**

A synchronization problem where multiple readers can access a shared resource simultaneously, but writers require exclusive access.



**Key Rules:**

* Multiple readers allowed concurrently
* Writers require exclusive access



**Key Challenges:**

* Managing access without conflicts
* Preventing starvation



**Solution Types:**

* Reader-priority (writers may starve)
* Writer-priority (readers may starve)
* Fair/balanced approach



**Core Insight:**

Synchronization design involves choosing policies that balance performance and fairness.



**Outcome:**

* Understood a real-world synchronization pattern
* Learned impact of priority decisions
* Strengthened applied system thinking
