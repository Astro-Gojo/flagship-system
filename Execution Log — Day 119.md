**Session Type:** Core Session



**Topic:** Throughput vs Latency



**Concept:**

Latency is the time taken for a single task, while throughput is the number of tasks completed per unit time.



**Key Learnings:**

* Throughput and latency represent different performance metrics
* Batch processing increases throughput but may increase latency
* Improving one metric can negatively impact the other



**Core Insight:**

System performance involves trade-offs between latency and throughput.



**Correction Made:**

* Clarified that batch processing primarily increases throughput, not both metrics simultaneously



**Outcome:**

* Understood fundamental performance trade-off
* Improved ability to reason about system efficiency
* Strengthened performance-oriented thinking
