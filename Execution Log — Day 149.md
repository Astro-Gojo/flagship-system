**Session Type:** Core Session



**Topic:** Concurrency vs Parallelism



**Concept:**

Concurrency involves managing multiple overlapping tasks, while parallelism involves executing multiple tasks simultaneously using multiple cores/processors.



**Key Learnings:**

* Concurrency and parallelism are related but different
* Concurrency can occur on a single core using task switching
* Parallelism requires simultaneous execution capability
* Concurrency improves coordination/responsiveness, while parallelism improves execution speed



**Core Insight:**

Concurrency focuses on coordinating tasks, while parallelism focuses on executing tasks simultaneously.



**Outcome:**

* Strengthened understanding of multitasking models
* Connected scheduling with multi-core execution
* Improved reasoning about scalability and system architecture trade-offs
