**Session Type:** Core Session



**Topic:** Deadlock Detection \& Recovery



**Concept:**

Deadlock detection identifies circular wait conditions, and recovery breaks the deadlock after it occurs.



**Detection Mechanism:**

* Use wait-for graph
* Cycle in graph indicates deadlock



**Recovery Methods:**

* Terminate one or more threads
* Preempt resources
* Rollback to previous state



**Key Learnings:**

* Detection \& recovery handle deadlocks after occurrence
* These methods involve trade-offs like overhead and data loss
* Used when prevention or avoidance is not feasible



**Core Insight:**

Detection and recovery are last-resort strategies in system design.



**Outcome:**

* Completed full understanding of deadlock handling approaches
* Learned trade-offs between different strategies
* Strengthened system-level decision-making ability
