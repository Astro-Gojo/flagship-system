**Session Type:** Core Session



**Topic:** Preemptive vs Non-Preemptive Scheduling



**Concept:**

Preemptive scheduling allows the CPU to interrupt a running thread, while non-preemptive scheduling allows a thread to run until completion or blocking.



**Key Learnings:**

* Non-preemptive scheduling is simpler but less responsive
* Preemptive scheduling improves responsiveness and fairness
* Preemption introduces overhead due to context switching



**Core Insight:**

Scheduler control over execution determines system responsiveness and performance.



**Outcome:**

* Understood execution control models
* Strengthened understanding of scheduling behavior
* Connected scheduling with real-world system responsiveness
