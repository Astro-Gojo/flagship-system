**Day 126 — Execution Log | Date : 26/04/2026**



**Session Type:** Core Session



**Topic:** Multi-Level Queue Scheduling



**Concept:**

Processes are divided into separate queues based on type, each with its own scheduling policy.



**Key Learnings:**

* Different types of tasks require different scheduling strategies
* Each queue can have its own algorithm
* CPU is shared between queues based on priority



**Key Issue:**

* Lower-priority queues may suffer starvation



**Core Insight:**

Real systems handle diverse workloads by organizing them into multiple scheduling layers.



**Outcome:**

* Understood multi-level scheduling structure
* Connected scheduling with workload classification
* Strengthened system design thinking
