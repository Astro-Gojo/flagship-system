**Day 135 — Execution Log** **| Date : 05/05/2026**



**Session Type:** Core Session



**Topic:** Starvation vs Deadlock



**Concept:**

Deadlock causes system-wide halt due to circular waiting, while starvation causes certain tasks to wait indefinitely due to unfair scheduling.



**Key Learnings:**

* Deadlock stops system progress completely
* Starvation allows system to run but affects fairness
* Aging can prevent starvation
* Different problems require different solutions



**Core Insight:**

System design must ensure both correctness and fairness.



**Outcome:**

* Clearly distinguished between deadlock and starvation
* Strengthened system failure analysis
* Improved understanding of fairness in scheduling
