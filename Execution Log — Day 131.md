**Session Type:** Core Session



**Topic:** Latency vs Throughput (System-Level View)



**Concept:**

Latency measures time per task, while throughput measures tasks per unit time. These are interconnected and often involve trade-offs.



**Key Learnings:**

* Batch processing increases throughput but increases latency
* Immediate processing reduces latency but lowers throughput
* Bottlenecks affect both latency and throughput
* System optimization requires trade-off decisions



**Core Insight:**

System design involves balancing latency and throughput based on application needs.



**Outcome:**

* Understood trade-offs in system performance
* Connected multiple system concepts into one model
* Strengthened high-level system design thinking
