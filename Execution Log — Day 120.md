**Session Type:** Core Session



**Topic:** Concurrency vs Parallelism



**Concept:**

Concurrency involves managing multiple tasks in progress (interleaved), while parallelism involves executing multiple tasks simultaneously.



**Key Learnings:**

* Concurrency does not require multiple cores
* Parallelism requires simultaneous execution
* Parallelism is a subset of concurrency



**Core Insight:**

Concurrency is about structure; parallelism is about execution.



**Outcome:**

* Clearly distinguished between two commonly confused concepts
* Strengthened understanding of system execution models
* Improved conceptual clarity for real-world applications
